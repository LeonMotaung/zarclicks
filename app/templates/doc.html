1nFloU API Documentation

Integrate influencer marketing tools into your applications with our comprehensive API. For complete details, visit API Full Doc.

1. Introduction

Overview

The 1nFloU API enables developers to integrate influencer marketing capabilities, including user management, influencer discovery, campaign creation, content collaboration, analytics, and payment processing. It is designed for scalability, security, and ease of use.

  • Purpose: Facilitate programmatic management of influencer marketing campaigns.
  • Features: User authentication, influencer search, campaign management, analytics, content collaboration, and secure payments.

Base URL

https://api.1nflou.com/v1/

Authentication Methods

The API supports multiple authentication methods to secure requests:

  • API Keys: Simple keys for basic access.
  • OAuth 2.0: Token-based authorization for third-party access.
  • JWT: Stateless tokens for microservices and user authentication.

Supported Formats

The API primarily uses JSON for request and response payloads. XML is supported for specific endpoints (specify Accept: application/xml in headers).

Rate Limits

100 requests per minute per user. See Section 5 for details on rate limit headers.

2. Authentication

Generate an API Key

API keys can be generated via the developer dashboard at API Full Doc. Log in, navigate to "API Keys," and click "Generate Key."

Example Request

curl -X GET "https://api.1nflou.com/v1/influencers" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"

Token Expiration and Refresh (OAuth 2.0)

Access tokens expire after 1 hour. Use refresh tokens to obtain new access tokens:

curl -X POST "https://api.1nflou.com/v1/oauth/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=refresh_token" \
  -d "refresh_token=YOUR_REFRESH_TOKEN" \
  -d "client_id=YOUR_CLIENT_ID" \
  -d "client_secret=YOUR_CLIENT_SECRET"

Example Error Response (Unauthorized)

{
  "error": true,
  "code": 401,
  "message": "Invalid API key"
}

3. Endpoints

Users

POST /users/register

Register a new brand or influencer.

curl -X POST "https://api.1nflou.com/v1/users/register" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "securepassword",
    "role": "brand"
  }'

POST /users/login

Authenticate and obtain a token.

curl -X POST "https://api.1nflou.com/v1/users/login" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "securepassword"
  }'

GET /users/me

Fetch current user profile.

curl -X GET "https://api.1nflou.com/v1/users/me" \
  -H "Authorization: Bearer YOUR_API_KEY"

Influencers

GET /influencers

List or search influencers by filters (category, followers, engagement, location).

curl -X GET "https://api.1nflou.com/v1/influencers?category=fashion&followers_min=10000" \
  -H "Authorization: Bearer YOUR_API_KEY"

GET /influencers/{id}

Get detailed influencer profile and stats.

curl -X GET "https://api.1nflou.com/v1/influencers/123" \
  -H "Authorization: Bearer YOUR_API_KEY"

POST /influencers

Add a new influencer (admin-only).

curl -X POST "https://api.1nflou.com/v1/influencers" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Doe",
    "category": "fitness",
    "followers": 50000
  }'

Campaigns

POST /campaigns

Create a new campaign.

curl -X POST "https://api.1nflou.com/v1/campaigns" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Summer Fashion Campaign",
    "start_date": "2025-10-01",
    "budget": 5000,
    "influencer_ids": ["123", "456"]
  }'

GET /campaigns

List all campaigns.

curl -X GET "https://api.1nflou.com/v1/campaigns" \
  -H "Authorization: Bearer YOUR_API_KEY"

GET /campaigns/{id}

Get campaign details.

curl -X GET "https://api.1nflou.com/v1/campaigns/789" \
  -H "Authorization: Bearer YOUR_API_KEY"

PUT /campaigns/{id}

Update a campaign.

curl -X PUT "https://api.1nflou.com/v1/campaigns/789" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Updated Summer Campaign",
    "budget": 6000
  }'

DELETE /campaigns/{id}

Delete a campaign.

curl -X DELETE "https://api.1nflou.com/v1/campaigns/789" \
  -H "Authorization: Bearer YOUR_API_KEY"

Content & Collaboration

POST /campaigns/{id}/content

Upload influencer content for approval.

curl -X POST "https://api.1nflou.com/v1/campaigns/789/content" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "content_url": "https://example.com/post.jpg",
    "caption": "Summer vibes!"
  }'

GET /campaigns/{id}/content

List campaign posts.

curl -X GET "https://api.1nflou.com/v1/campaigns/789/content" \
  -H "Authorization: Bearer YOUR_API_KEY"

POST /messages

Send messages between brand and influencer.

curl -X POST "https://api.1nflou.com/v1/messages" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "recipient_id": "123",
    "message": "Looking forward to the campaign!"
  }'

Analytics

GET /analytics/campaign/{id}

Retrieve campaign performance metrics (impressions, engagement, conversions).

curl -X GET "https://api.1nflou.com/v1/analytics/campaign/789" \
  -H "Authorization: Bearer YOUR_API_KEY"

GET /analytics/influencer/{id}

Retrieve influencer performance stats.

curl -X GET "https://api.1nflou.com/v1/analytics/influencer/123" \
  -H "Authorization: Bearer YOUR_API_KEY"

4. Error Handling

Standardized Error Response Format

{
  "error": true,
  "code": 401,
  "message": "Invalid API key"
}

Common Error Codes

  • 400 Bad Request: Invalid request parameters or payload.
  • 401 Unauthorized: Invalid or missing authentication credentials.
  • 404 Not Found: Resource not found.
  • 429 Too Many Requests: Rate limit exceeded.
  • 500 Internal Server Error: Server-side issue.

5. Rate Limits

The API enforces a rate limit of 100 requests per minute per user. Responses include the following headers:

  • X-RateLimit-Limit: Maximum requests allowed (100).
  • X-RateLimit-Remaining: Remaining requests in the current window.
  • X-RateLimit-Reset: Unix timestamp when the limit resets.

Example Rate Limit Exceeded Response

{
  "error": true,
  "code": 429,
  "message": "Rate limit exceeded. Try again later."
}

7. SDKs / Examples

Curl

curl -X GET "https://api.1nflou.com/v1/influencers?category=fashion" \
  -H "Authorization: Bearer YOUR_API_KEY"

JavaScript (Axios)

const axios = require('axios');

async function getInfluencers() {
  try {
    const response = await axios.get('https://api.1nflou.com/v1/influencers?category=fashion', {
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY'
      }
    });
    console.log(response.data);
  } catch (error) {
    console.error('Error:', error.response.data);
  }
}

Python (Requests)

import requests

url = "https://api.1nflou.com/v1/influencers?category=fashion"
headers = {"Authorization": "Bearer YOUR_API_KEY"}

response = requests.get(url, headers=headers)
if response.status_code == 200:
    print(response.json())
else:
    print(f"Error: {response.json()}")

PHP (Guzzle)

use GuzzleHttp\Client;

$client = new Client();
$response = $client->request('GET', 'https://api.1nflou.com/v1/influencers?category=fashion', [
    'headers' => [
        'Authorization' => 'Bearer YOUR_API_KEY',
    ]
]);

if ($response->getStatusCode() == 200) {
    echo $response->getBody();
} else {
    echo 'Error: ' . $response->getBody();
}
